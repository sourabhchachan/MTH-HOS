<analysis>**original_problem_statement:**
The user wants to build a comprehensive, order-driven hospital operations system called MTH Hospital Operations System.

**PRODUCT REQUIREMENTS:**

-   **System Philosophy:** Everything is an order, following a universal workflow: Order Creation → Automated Routing → Order Dispatch → Order Receipt → Order Completion. Each stage requires automatic time-stamping and user-stamping for a complete audit trail. The system must not have hard-coded logic.
-   **Core Modules:** Patients, Users, Departments, Items (Item Master), Orders (including Returns), Vendors, Assets, HR System (Attendance, Payroll), Reporting, Billing, and an Insight Engine.
-   **Permissions & Users:** Department-driven (not role-based) permissions controlled by department memberships and Item Master configurations. Login via phone number with JWT-based authentication.
-   **Order & Item Logic:** Must support partial dispatch, order priorities, patient-linked items, and return orders. Item Master controls routing and ordering permissions.
-   **Billing & Finance:** Billing is generated only when an order status is 'Completed'.
-   **UI/UX:** Must be a mobile-first Progressive Web App (PWA) with minimal, task-focused dashboards.
-   **Admin Tools:** Admins must be able to populate all master data, including a bulk CSV upload for items.
-   **Latest Request:** The user has asked for an Operational Workflow Simulation mode to test the core engine under real-world conditions.

**User's preferred language**: English
