The user wants to build a comprehensive, order-driven hospital operations system called MTH Hospital Operations System.

**PRODUCT REQUIREMENTS:**

**System Philosophy:**
- Everything is an order.
- The universal workflow is: Order Creation → Automated Routing → Order Dispatch → Order Receipt → Order Completion.
- Every stage must be automatically time-stamped and user-stamped for a complete audit trail.
- The system must not have hard-coded logic.

**Core Modules:**
- Patients (UHID + IPD), Users/Employees, Departments, Items (Item Master), Orders (including Returns), Vendors, Assets, HR System (Attendance, Payroll), Reporting, Billing, and an Insight Engine.

**Patient Workflow:**
- Fixed phases: Pre-Admission, Admission, IPD, Discharge, Post-Discharge, Archived.
- These phases are for analytics and tracking ONLY and must not restrict ordering.

**Permissions & Users:**
- The system is department-driven, NOT role-based.
- User permissions are controlled by their primary/secondary department memberships and Item Master configurations.
- Login ID is the user's phone number, using JWT-based authentication.

**Order & Item Logic:**
- **Order Status:** Created, Partially Dispatched, Fully Dispatched, Completed, Cancelled.
- **Order Item Status:** Pending Dispatch, Partially Dispatched, Fully Dispatched, Received, Cancelled.
- **Partial Dispatch:** Must be supported. A single order item can have multiple dispatch events.
- **Item Master Routing:** An item has a single . Any department can order an item, and the order is auto-routed to the  queue.
- **Item Ordering Permissions:** The  must specify which departments are allowed to order an item (All Departments or a select list).
- **Priority System:** Orders can be 'Normal' or 'Urgent'. An order cannot mix priorities.
- **Patient Linkage:** Items can be configured to require a patient IPD number (for Active, Inactive, or Both IPD statuses).
- **Return Orders:** Must be supported, referencing the  and .

**Billing & Finance:**
- Billing is generated ONLY when an order status is 'Completed'.
- Bill Amount =  × .
- Item cost and vendor details are visible only to Admins or authorized users.
- Each item must be mapped to a vendor in the .

**UI/UX:**
- The system must be mobile-first, designed as a Progressive Web App (PWA).
- User dashboards should be minimal and task-focused (Pending tasks, Create Order, Dispatch Queue, Receive Confirmation).
- Admin dashboards can be more complex with reporting.
- Admin must be able to populate master data (Departments, Users, Items, etc.) through a setup interface, including bulk CSV upload for items.

**System Scale:**
- Support up to 150 concurrent users.
